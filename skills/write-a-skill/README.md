# Write a Skill

Create new agent skills with proper structure, progressive disclosure, and clear triggers for use in the Privacy Skills project.

## When to use this

- You want to add a new skill to the Privacy Skills collection.
- You need to write a SKILL.md file and README.md for a new privacy, security, or governance topic.
- You want to follow the consistent structure used across this project.

## The checklist

1. Define the skill's purpose: what privacy, security, or data risk does it address?
2. Write a one-sentence description that tells an agent when to invoke the skill.
3. Draft the SKILL.md with: frontmatter, quick start, workflow, checklist, output format, and disclaimer.
4. Keep SKILL.md under 100 lines where possible.
5. Draft the README.md with: skill name, summary, when to use, checklist, examples, related skills, related questions, disclaimer, and Berrysbay CTA.
6. Check that the description in SKILL.md frontmatter includes "Use when..." with specific triggers.
7. Check that the checklist is numbered and actionable.
8. Check that related skills and related questions link to existing files in the project.
9. Check that the tone is practical, plain-language, and not legalistic.

## Examples

- A new "cookie consent review" skill is created to help teams check cookie banners and consent flows.
- A "data subject access request handler" skill is written to guide teams through responding to access requests.
- A "AI model card privacy review" skill is drafted for teams publishing or evaluating AI models.

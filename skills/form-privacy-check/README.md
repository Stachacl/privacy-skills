# Form Privacy Check

Review a form, survey, or sign-up flow for unnecessary or risky data collection before publishing it.

## When to use this

- You are designing a new form that collects personal information and want to check it before launch.
- You are auditing an existing form to see whether it asks for more than is needed.
- You are adding a new field to a form and want to confirm it is justified and handled safely.

## The checklist

1. List every field in the form.
2. For each field, ask: why is this needed, and what happens if it is left blank?
3. Check whether any required fields could be optional instead.
4. Check whether any fields collect sensitive data (health, financial, government IDs, children's information).
5. Check whether free-text fields might invite people to share more than you need.
6. Check whether the form explains how the data will be used.
7. Check whether the form explains who will see the data.
8. Check whether there is a privacy notice or link to one.
9. Check whether the form is transmitted and stored securely.
10. Check how long submitted data is kept and who can access it.

## Examples

- A newsletter sign-up form asks for date of birth, phone number, and job title when only an email address is needed.
- A feedback form includes a free-text "tell us more" field that regularly receives sensitive personal disclosures.
- A registration form for a community event asks for dietary requirements alongside names and addresses, stored in a shared spreadsheet with no access controls.

# Approved AI Tools Register

Build and maintain a register of AI tools used across your organisation, with approval status, allowed uses, data restrictions, owner, and review date.

## When to use this

- Your team uses multiple AI tools and you want a clear record of what is approved and what is not.
- You need to manage AI tool sprawl or shadow AI use across departments.
- You are building an AI governance process and need a central register to start from.

## The checklist

1. List all AI tools currently in use, requested, or under consideration.
2. For each tool, document the approval status: approved, restricted, not approved, or under review.
3. Record allowed uses and what data types must not be shared with each tool.
4. Note whether each tool is a consumer product or an enterprise service with data protection terms.
5. Check whether prompts or inputs are used for model training.
6. Assign an owner and a review date for each entry.
7. Identify any tools being used without formal approval (shadow AI).

## Examples

- An approved AI coding assistant is listed with a note that credentials and customer data must not be included in prompts.
- A consumer image generator is marked "not approved" because it uses uploaded images for training and has no enterprise tier.
- A new AI summarisation tool requested by the sales team is listed as "under review" while vendor terms are checked.

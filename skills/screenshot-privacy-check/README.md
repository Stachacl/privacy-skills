# Screenshot Privacy Check

Review a screenshot for personal information, sensitive data, or identifying details before sharing, posting, or submitting it.

## When to use this

- You are about to share a screenshot in a support ticket, blog post, presentation, or documentation.
- You want to check whether a screenshot contains information that should be blurred or removed.
- You are preparing screenshots for public-facing content and want to make sure they are safe.

## The checklist

1. Check for visible names, email addresses, phone numbers, or physical addresses.
2. Check for profile photos or avatars of real people.
3. Check for account usernames, user IDs, or customer reference numbers.
4. Check for financial information: balances, card numbers, invoices.
5. Check for health or medical information.
6. Check for government or official ID numbers.
7. Check for internal system names, URLs, or environment details that should not be public.
8. Check for database IDs, API keys, or tokens.
9. Check for confidential business data or internal documents visible on screen.
10. Check for other people's messages or conversations.
11. Check for browser history, tabs, or bookmarks visible in the background.
12. For each issue found: blur, crop, replace with placeholder, or recreate with dummy data.

## Examples

- A developer shares a screenshot to report a bug. The screenshot shows the production dashboard URL and a customer's email address in the current user field.
- A support agent shares a screenshot with a vendor. The sidebar of the application shows other customers' names.
- A presentation slide contains a screenshot of an internal tool with real employee names and salaries visible.

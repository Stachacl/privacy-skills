# Cloud Storage Sharing Check

Review cloud file and folder sharing for privacy, access control, oversharing, and retention risks before sharing files through Google Drive, OneDrive, Dropbox, SharePoint, iCloud, or similar tools.

## When to use this

- You are about to share a file or folder link and want to check the access settings.
- You are reviewing how your team shares documents internally and with external parties.
- You want to check whether old shared links are still active and accessible.

## The checklist

1. Identify what is being shared: file, folder, or a shared drive.
2. Check who the link is shared with: specific people, anyone with the link, or public.
3. Confirm the recipient actually needs access to the full document or whether a subset is enough.
4. Check whether the link has an expiry date, or whether access is permanent.
5. Check whether the recipient can edit, comment, or only view.
6. Identify what personal or sensitive information is in the file or folder.
7. Check whether the folder contains other files the recipient should not see.
8. Review whether old shared links should be revoked.
9. Consider whether a more controlled sharing method (e.g. a portal or encrypted transfer) is more appropriate.

## Examples

- A HR folder shared with a payroll contractor includes subfolders containing disciplinary records that the contractor does not need.
- A public "anyone with the link" setting was set during a quick internal review and never changed back.
- A shared Google Drive link sent to a client six months ago is still active and the drive now contains newer confidential documents.

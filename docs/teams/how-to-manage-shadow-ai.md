# How to manage shadow AI

## Short answer

Shadow AI — staff using AI tools that have not been reviewed or approved — is common, and banning tools outright rarely works. A more effective approach is to make the approved path easy, maintain a clear register of what is and is not allowed, and give staff a way to request new tools quickly.

## What to check

1. Find out what AI tools staff are currently using — including free tools, browser extensions, and integrations with existing software.
2. Check whether any of these tools are accessing customer data, internal documents, or HR information.
3. Check whether staff are using consumer AI tools for tasks that involve sensitive data.
4. Build an approved AI tools register with clear status for each tool: approved, restricted, not approved, or under review.
5. Create a simple process for staff to request approval for new tools before using them.
6. Make the approved path convenient — if approved tools are harder to use than unapproved ones, staff will use unapproved ones.
7. Create a brief AI use policy that explains what is and is not allowed, and why.
8. Build regular review into the process: AI tools change quickly, so approvals should be revisited periodically.
9. Treat violations as a training opportunity first, not a disciplinary issue, unless the risk is serious.

**Common shadow AI patterns to look for:**

- Consumer AI writing tools used to draft customer communications
- AI note-takers added to meetings without team-wide awareness
- Browser AI extensions with access to all pages, including internal tools
- AI plugins in productivity tools (email, calendar, spreadsheets) that upload content to vendor servers
